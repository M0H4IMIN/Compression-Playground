How RLE works

RLE replaces a run of identical characters with a (count, char) pair. Best for data with long repeated runs — like bitmap images or fax data. It can make things worse for varied text.

How Huffman works

Huffman builds a binary tree: frequent chars get short codes, rare chars get long codes. The code table is built fresh per input — it's provably optimal for that specific character distribution.

How LZ77 works

LZ77 maintains a sliding window. For each position it looks back in the buffer for the longest match. A match emits (offset, length, next char). This is the basis of gzip, deflate, and zstd.
